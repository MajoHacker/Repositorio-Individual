Preguntas laboratorio 1:

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una inmensa red de computadoras conectadas entre sí, mientras que la World Wide Web es un servicio basado en Internet que permite el acceso a información mediante vínculos.

¿Cuáles son las partes de un URL?


1. Protocolo: permite la transferencia de información en Internet
2. Subdominio: objetivo de crear divisiones dentro del sitio
3. Dominio: hace referencia al nombre único que identifica a un sitio web
4. TLD: indica la naturaleza del dominio
5. Carpeta o subcarpeta: organizar la estructura de una página web
6. Página: hace referencia a un archivo del sitio web
7. Etiqueta: referencia interna al contenido de una página

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: solicita datos de un recurso específico.
HEAD: obtiene metadatos sobre un recurso.
POST: enviar datos o crear nuevos recursos.
PUT: actualiza un recurso existente.
PATCH: realizar modificaciones parciales en un recurso existente.
DELETE: suprime un recurso

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

POST, ya que este método está diseñado para enviar datos desde el cliente, es decir, desde el navegador web.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET, ya que este solicita los datos de un recurso específico en un servidor web.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Si aparece un código 200, esto quiere decir "OK", lo cual nos indica que el proceso se recibió, procesó y realizó correctamente.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, ya que el error 404 nos indica que el servidor no pudo encontrar el recurso solicitado, por lo que el desarrollador deberá verificar en su sitio web que todos los recursos sean visibles.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, ya que el error 500 nos indica un error interno en el servidor, por lo que el desarrollador deberá encontrar dicho error y corregirlo.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que el atributo es obsoleto, por lo que se recomienda no usarlo en el futuro. Algunos ejemplos son: font, center, ya que actualmente se pueden usar estos atributos más fácilmente desde el CSS.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 incluye nuevos elementos semánticos, soporte para audio y video sin necesidad de plugins de terceros, almacenamiento sin conexión, compatibilidad con dispositivos móviles, nuevas APIs y etiquetas.

¿Qué componentes de estructura y estilo tiene una tabla?

ESTRUCTURA:
- table: define la tabla.
- tr: define una fila.
- th: define una celda de encabezado en una fila o columna.
- td: define una celda de datos en una fila.

ESTILO:
- border: especifica el ancho y el estilo del borde de la tabla y sus celdas
- background-color: especifica el color de fondo de la tabla
- padding: especifica el espacio entre el contenido de las celdas y sus bordes.
- color: especifica el color del texto dentro de la tabla.

¿Cuáles son los principales controles de una forma HTML5?

select: permite al usuario seleccionar una opción de una lista desplegable.
checkbox: casilla de verificación que permite al usuario seleccionar una o varias opciones de una lista.
number: campo de entrada numérico que solo permite ingresar números.
size: permite establecer la longitud de los controles de texto
email: campo de entrada de texto diseñado específicamente para ingresar direcciones de correo electrónico
password: el texto ingresado se muestra como asteriscos o puntos para ocultar la contraseña.

¿Cuál es el ciclo de vida de los sistemas de información?

Es un marco de trabajo que describe las etapas por las que pasa un sistema, desde su diseño hasta su desarrollo, implementación, operación y mantenimiento.

¿Cuál es el ciclo de desarrollo de sistemas de información?

Preguntas laboratorio 3:

¿Cómo ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

No suele usarse mucho, sin embargo cuando estas en un proyecto donde deseas que un atributo tenga más importancia sobre otro, este se vuelve muy útil ya que permite declarar un atributo de forma que este tenga prevalencia sobre otras declaraciones posteriores

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Debido a que la imagen de fondo puede contener colores que afecten la legibilidad del sitio, además imagenes de fondo pesadas pueden aumentar significativamente el tiempo de carga de la página, lo que generalmente lleva a una mala expereiencia del usuario.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Considero que son muy importantes ya que pueden simplificar el trabajo, sin embargo su uso depende totalmente del contexto y los requisitos de diseño del proyecto.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Principalmente porque al reducir el código, se eliminan espacios en blanco, comentarios y otros caracteres no esenciales, lo que hace que el tiempo de descarga sea mucho más rapido.
